# 配对样本 · Pattern C 扫除（"回收装置失灵了" → "风有脾气"）

> 失败模式名：**Pattern C——把 Anna 当故障物诊断。** Edmund 的回应只允许 Pattern A（shared delight）或 Pattern B（荒诞幽默），Pattern C 失效，必扫除（宪法 §3.4；prose-gate 在册）。

## 失败版（旧版，仅存一句引文）

> "回收装置失灵了"

⚠ 公理一现场标本：旧版完整文本**当时没存，已永远丢失**——只剩 repo-plan 公理二里引用的这一句。失败点（据宪法重建）：风筝飞丢时 Edmund 的第一反应是诊断**装置**——把事故翻译成故障清单，把女儿当报修单处理。机械正确，情感为零，世界死的。

## 修复版（现行 canon 素材，原样）

> 安娜没有哭。她盯着那棵树看了整整五分钟，然后转头对埃德蒙说："爸爸，我需要一架能飞回来的帆。"
>
> 埃德蒙蹲下来，和她一起看挂在下面的那个银色小点。他严肃地点了点头。
>
> "那阵风脾气不好。"
>
> "风有脾气？"
>
> "今天的风有。我们惹它了。"他站起来，拍了拍膝盖上的灰。"走，我们想办法把它弄回来。"

（`30-design/modules/sensory-world-3-6.md` 1.2，含设计备注：Edmund 的 first move 不是诊断 Anna，是和她一起面对问题 + 把风拟人化——Pattern B。"风有脾气"同时是 echo 链种子：9 岁扩展成"崖壁有脾气"。）

## 为什么（据宪法转述，非作者原话）

修复做了三件事：①蹲下来**一起看**——共面对，不是上对下处理；②"风有脾气"把物理事件翻译成一个六岁孩子能拿在手里的世界观——且这个拟人不是哄小孩，是 Edmund 自己真实的机械观（机器是有脾气的活物）；③顺手种下一条九年后才结果的 echo 链。一句台词同时完成情感、世界观、伏笔三层——这是 Pattern B 与 Pattern C 之间隔着的整个距离。

## 自检口令

写任何 Edmund 对话前问：他这句话是在**修 Anna**，还是在**和 Anna 一起看世界**？前者删。
