# 60-taste · 评点层

> **本层性质声明：这里的一切是接班人的审美训练基底。**
>
> 设定教它世界是什么，本层教它世界该是什么感觉。品味不靠规范文档传承，靠文本边上的朱批传承——脂砚斋批红楼、金圣叹批水浒的评点本传统。本层是永久层，不是脚手架；公理一在此执行：**每一次否决、修改、批注都必须沉淀，当时没记就永远丢失。**

## 四个子目录

### `rulings/` — 审美裁决

一裁一文件，`0001-<slug>.md` 四位递增。格式：

```
引文（被否版本，原样）
→ 改后版本
→ 作者的为什么（原话优先；AI 转述需标注）
```

### `paired-samples/` — 失败/修复配对样本库

品味传承最密集的形式。每对 = 失败版 + 修复版 + 一行原因。来源：prose-gate 自检修复（持续生产）、宪法既有配对（Pattern C 旧版"回收装置失灵了"→"风有脾气"，待 Phase 3a 抽取）、废弃案例（M-09 Thread Festival 等）。

### `reveries/` — 原浆感受

作者未经整理的白日梦式感受倾倒。**规则：原样保存，不润色、不结构化、只标日期。** 文件名 `YYYY-MM-DD-<slug>.md`。front-matter 标 `source: author | ai-approved`（作者原话永远是更高权重的基准音）。待回收的已知存量：东部 outpost 黑熔岩附草段、"油渍快乐"等（目前只活在聊天记录里，Phase 3c）。

### `reviews/` — 朱批

作者对草稿的边注，按集归档（`ep-I-05.review.md`）。每条批注 = 定位引文 + 作者原话（**一字不改**）。PR review 结束后收割入此；重大否决升格为 rulings。
