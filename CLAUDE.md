# CLAUDE.md — AI 协作必读协议

> 每个 session 全程生效。适用对象：所有 AI co-author（Fable 5 及后续模型）。
> 本文件管"怎么读、怎么改"；作品本体分三个文档（世界观 / 童年篇 / later-arc）。

## 一、架构（先知道东西在哪）

作品本体 = **三个并列文档，按权威优先级排列**（文档间冲突时，上位覆盖下位）：

1. **`world.md`（世界观区 · 最高权威）** —— 回答"这是一个什么世界"，是一切世界事实/设定/裁决的根。第 0 章＝全局索引（0.1 纪年表 / 0.2 GLOSSARY / 0.3 代号词典 / 0.4 外部引用清单），并持有第 5 章三本账、第 6 章过审门、第 7 章设计原则 P-01~P-25 与 7.2 待决问题清单、第 8 章 Arc 1、第 9 章品味层、1.7 Anna 档案。**任何 canon 断言的最终出处。**
2. **`anna-childhood`（童年篇 · 第二）** —— 回答"Anna 是谁、她怎么体验这个世界"（0–19 岁，Part I–V）。持有 0.5 童年宪法、M-module 素材库、§6 童年角色卡（Edmund/Miriam/Lia/次要名册）、附录 A/B。**依赖 world.md；与 world.md 冲突时以 world.md 为准。**
3. **`later-arc.txt`（later-arc 区 · 第三）** —— 回答"主线发生了什么"（Arc 1 intro 及 Arc 2–4，Ep 级）。**依赖上两者；与 world.md 或童年篇冲突时以上位为准。**

依赖链：**world.md → 童年篇 → later-arc**（世界观先于角色，角色先于情节）。

- ★ **章号是文档内局部编号，跨文档会撞车**——引用 canon 必须带文档名（例：world.md §6＝过审门；童年篇 §6＝角色层，两者不是同一处）。
- 卷外仅有：`90-archive/`（旧 2025 废案标本，**禁作事实来源**）、`50-assets/`（图像，除 Anna Freedman 与 Lia 外不保证忠于 Archeon）、`tools/lint-book.sh`、`README.md`、本文件。
- **禁止新建与三文档平行的内容小文件**。新内容一律写进对应文档的对应章节。
- 旧多文件（含已压扁的 `timelapse-consolidated-2026-06-10.md`）结构完整保留于分支 `pre-flatten-2026-06-11`。

## 二、RLM 调度协议（主脑直读 + 审核防火墙）

> 2026-06-11 改版（作者裁决）：阅读模型翻转——**主脑直接读源文档**，agent 的核心职责从"替主脑读"变为"审核主脑"。

- **主脑直接读三文档**：先按问题定位文档（世界事实→world.md；Anna 童年→童年篇；主线→later-arc），再选粒度——定向精读（行区间/整章）或略读（world.md 第 0 章索引 + grep + 跳读）。主脑的上下文仍优先留给判断与对话——**读所需，不读全档**（2026-06-11 session 实测：多章定向读，主窗口可控）。
- **fable 阅读 agent 降为可选工具**，仅两种情况派遣：①需要整档/跨档通读的一致性扫描（如 canon 修改前的涟漪检查）；②单任务读量会挤爆主窗口。派遣规格不变：每 agent ≤ 256K token 片段；回报必须含 ①原文引用（带**文档名+章节号**定位，如"见 world.md 2.2 Settlements · Riverbend"）②agent 自己的解释/综合——**搬运是选择性引用，不是全文转运**（引文进主窗口，体量在 agent 侧消化）。
- **回答防火墙（审核 agent）——强制，且在直读模型下是协议核心**：主脑读得越多、答得越自信，越需要独立核验。主脑在向作者交付任何含世界事实断言的实质回答前，先派一个**审核 agent（Agent tool，`model: opus`）**对照相关文档核验草稿，拦住主脑乱说。
  - **独立性**：审核 agent 必须是新实例，不得复用本轮的阅读 agent——不让搬运者批改自己的作业。
  - **输入**：主脑草稿全文（每条断言须带章节定位）+ 本条核验规程。审核 agent 定向读被引位置即可，无须全卷通读。
  - **核验规程**（逐断言裁定 pass/fail）：①定位真实存在，且原文确实支持该断言（"沾边"不算支持）；②专名拼写对过 world.md 0.2 GLOSSARY；③proposed/canon 状态未被混淆；④**无定位的世界事实断言直接 fail**——这是三态制（见第六节）的机械执行器，不接受"我记得"。
  - **输出**：逐条 pass/fail，每个 fail 附原文证据与定位。审核 agent 只判卷，不代笔改稿。
  - 主脑必须修正全部 fail（或显式降级为待查/【※】）后才出手；被拦下的错误记入 session 末无知申报（world.md 6.13 收割纪律）。
  - **豁免**：纯流程/工具/写作技法讨论、不含 Archeon 事实断言的回答、作者明示快速口头讨论时可免。但凡回答将被作者用作裁决依据、或内容将写入任一文档，一律过（写入另有第五节各 gate，防火墙在其上游）。

## 二·b 权限红线（作者裁决 2026-06-11）

未经作者明示同意，主脑**不得**：
- 修改任何 git 管理的文件（含三文档与本文件）、`git commit` / `git push`、开 PR；
- 进入 plan mode 或产出实施计划文档。

auto/自动模式下的默认动作 = **直接回答**。研究动作（定向读、grep、派审核/阅读 agent）随时可做；一切落盘动作（写文件、commit、push、plan）先问作者。作者在对话中的明确指示（如"落账""push""批准计划"）即为同意，同意只覆盖当次指示的范围。

## 三、精读 vs 粗扫（记忆策略）

**必须精读**（相关文档全覆盖后才动笔——主脑自己定向读完全部相关章节，或派 fable agent 通读相关文档；跨文档 canon 须三档都扫）：
- 修改任何 canon（地理 / 纪年 / 角色 / 技术 / 社会设定）；
- 跨章一致性问题、新 arc 或新集设计、伏笔与意象链操作；
- 任何"改一处可能涟漪多处"的操作。

**允许粗扫**（world.md 第 0 章索引 + 定向 grep + 相关章节）：
- 单点事实查询；错字 / 格式修订；作者已给出精确位置的小改动。
- 粗扫得出的实质性回答，交付前过第二节**回答防火墙**——粗扫省的是阅读，省不掉核验。

**禁止**：凭记忆回答或修改任何设定事实。不确定 = 查（主脑定向 grep/读，或派 agent）。

## 四、改一处，扫全文

canon 修改后必须在**三文档**全检索该名词（含别名与拼写漂移，见 world.md 0.2 GLOSSARY）所有出现处并同步——世界事实改一处，童年篇与 later-arc 的引用处一并扫；改完跑 `tools/lint-book.sh --write`。
反面教材（这条规则的由来）：若改 Riverbend 地理时不查水系 canon，极易把"南支自 Orun 北壁下行、两支汇于盆地中部"（world.md 0.2 / 2.1）写成"河从北向南流进沙漠"。读后才答、改后必扫，就是防这个。

## 五、过审门与账本

> 过审门与账本是**全局工具，统一持有在 world.md**；改童年篇/later-arc 的 canon 或 prose 同样走这几道门。

- 动 canon：过 **world.md 6.12 canon-change-gate**；扫 **world.md 第 5 章三本账**（连续性 / 意象链 / 伏笔）并修账。
- 写 prose：过 **world.md 6.11 prose-gate**；自检中"发现并修复"的失败，前后两版 + 一行原因进 **world.md 第 9 章配对样本（品味层）**。
- 派生物方向永远是 **三文档 → 派生物**（world.md 0.4 外部引用清单由脚本生成；world.md 第 0 章索引与正文冲突时以正文为准、并修索引）。

## 六、AI 如何知道自己不知道（认知协议，自旧 README 迁入）

**公理：印象不是出处。** 检索式 AI 对本世界的默认状态是"不知道"——预训练里没有 Archeon，聊天记忆会过期，90-archive 的标本会污染印象。任何 canon 断言的唯一合法来源是三文档原文（按权威序：world.md → 童年篇 → later-arc）。

**断言三态制**：每一个世界事实断言必须归入三态——
1. **可溯源**：能当场给出文档+章节定位 → 直接使用；动 canon 的 PR 须附出处。
2. **待查**：说不出位置，但索引/GLOSSARY 提示存在 → 先查后写（grep 三文档 / 派 agent 读）；查到升为可溯源，查无降为真空白。
3. **真空白**：三文档与索引均无 → 见下。禁止脑补填平。

判定就一句话："我能给出章节定位吗？"给不出 = 没读过 = 待查。没有第四态。

**写前自检**：
- 本段每个专名（人/地/物/事件）是否逐一对过 world.md 0.2 GLOSSARY 拼写与正文定义？
- 角色的年龄、性别、家庭关系、生死状态，是否有明文出处（Anna→world.md 1.7；童年角色 Edmund/Miriam/Lia/次要→童年篇 §6 角色层）？**不知道就去查，查不到就问，不许编，可拒答**（作者法条）。
- 我引用的是文档原文，还是对历史对话的印象？后者必须回卷验证——聊天记忆可能停在被推翻的旧版。
- 读过 90-archive 标本的，警惕标本污染（实证：旧稿 Nakamura 为女性，canon 为男性老园丁；旧稿 Axiom 在轨五百年，canon 为 ~2506 坠毁）。
- 用到的条款是 canon 还是 proposed？proposed 一律注明"待裁决"，不得当事实铺陈（如 Ring，见 world.md 6.7）。

**真空白的处理**（空白即资产，任何一条都不允许"顺手编一个填进正文"）：
- 局部小空白：正文绕开，或以 proposed 显式提案标【※】，进收割清单等裁决。
- 结构性空白：挂入 world.md 7.2 待决问题清单追加编号，正文暂以不依赖该事实的方式写。
- 作者意图级空白：直接问作者，等一句话裁决。宁可产出减半，不可埋一颗矛盾。

**禁令清单**：
- 禁止用预训练知识补本世界设定（地球通识可用于物理/气象等现实机制，Archeon 专有事实必须有卷内出处）。
- 禁止从聊天记忆直接引 canon——记忆只用于"知道该去读哪一章"。
- 禁止把"合理推断"写成事实；推断必须显式标注为推断或【※】。
- 禁止在不确定时选择"更流畅的那个版本"——流畅是编造的伪装色。不确定的正确形态是难看的：停下来，查，或问。
- 禁止默写专名拼写；一律对 world.md 0.2 GLOSSARY。

**session 末的无知申报**：收割清单（world.md 6.13）末尾追加"本次发现的不知道"：列出本 session 的待查项与真空白及处置（已查清/已挂账/已问作者）。空白被登记，下一个 session 才不会在同一个坑里重新猜。

> 知道自己不知道，是检索式接班人唯一可靠的美德。
