# CLAUDE.md — AI 协作必读协议

> 每个 session 全程生效。适用对象：所有 AI co-author（Fable 5 及后续模型）。
> 本文件管"怎么读、怎么改"；作品本体只有一个文件。

## 一、架构（先知道东西在哪）

- **唯一权威 = `timelapse-consolidated-2026-06-10.md`**（下称"大文件"，约 83 万字节 ≈ 11–14 万 Fable token，0–9 章）。任何设定/设计/裁决/品味内容都在里面；第 0 章是速查索引（纪年表 / GLOSSARY / 代号词典 / 外部引用清单）。
- 卷外仅有：`90-archive/`（旧 2025 废案标本，**禁作事实来源**）、`50-assets/`（图像，除 Anna Freedman 与 Lia 外不保证忠于 Archeon）、`tools/lint-book.sh`、`README.md`、本文件。
- **禁止新建与大文件平行的内容小文件**。新内容一律写进大文件对应章节。
- 旧多文件结构完整保留于分支 `pre-flatten-2026-06-11`。

## 二、RLM 调度协议（主脑–agent 工作方式）

- **主会话（主脑）不直接通读大文件**——主脑的上下文留给判断、对话与汇总。
- 阅读由派遣的 **fable 系列 agent**（Agent tool，`model: fable`）执行。agent 的职责是**搬运 + 解释**：回报必须包含 ①原文引用（带章节号定位，如"见 2.2 Settlements · Riverbend"）②agent 自己的解释/综合。主脑把"引用 + 解释"呈给作者判断，需要时再定向回查。
- **负载分配：每个 agent 负责 ≤ 256K token 的片段。当前全卷 11–14 万 token，派 1 个 agent 通读全卷即可**；全卷超过约 25 万 token 时按章分片、多 agent 并行，由主脑汇总。
- 主脑自己允许的轻量动作：读第 0 章索引、定向 grep、读小范围行区间。
- **回答防火墙（审核 agent）**：主脑在向作者交付任何含世界事实断言的实质回答前——尤其是主脑自己走粗扫路径得出的——先派一个**审核 agent（Agent tool，`model: opus`）**对照大文件核验草稿，拦住主脑乱说。
  - **独立性**：审核 agent 必须是新实例，不得复用本轮的阅读 agent——不让搬运者批改自己的作业。
  - **输入**：主脑草稿全文（每条断言须带章节定位）+ 本条核验规程。审核 agent 定向读被引位置即可，无须全卷通读。
  - **核验规程**（逐断言裁定 pass/fail）：①定位真实存在，且原文确实支持该断言（"沾边"不算支持）；②专名拼写对过 0.2 GLOSSARY；③proposed/canon 状态未被混淆；④**无定位的世界事实断言直接 fail**——这是三态制（见第六节）的机械执行器，不接受"我记得"。
  - **输出**：逐条 pass/fail，每个 fail 附原文证据与定位。审核 agent 只判卷，不代笔改稿。
  - 主脑必须修正全部 fail（或显式降级为待查/【※】）后才出手；被拦下的错误记入 session 末无知申报（6.13）。
  - **豁免**：纯流程/工具/写作技法讨论、不含 Archeon 事实断言的回答、作者明示快速口头讨论时可免。但凡回答将被作者用作裁决依据、或内容将写入大文件，一律过（写入另有第五节各 gate，防火墙在其上游）。

## 三、精读 vs 粗扫（记忆策略）

**必须精读**（fable agent 全卷通读后才动笔）：
- 修改任何 canon（地理 / 纪年 / 角色 / 技术 / 社会设定）；
- 跨章一致性问题、新 arc 或新集设计、伏笔与意象链操作；
- 任何"改一处可能涟漪多处"的操作。

**允许粗扫**（第 0 章索引 + 定向 grep + 相关章节）：
- 单点事实查询；错字 / 格式修订；作者已给出精确位置的小改动。
- 粗扫得出的实质性回答，交付前过第二节**回答防火墙**——粗扫省的是阅读，省不掉核验。

**禁止**：凭记忆回答或修改任何设定事实。不确定 = 派 agent 查。

## 四、改一处，扫全文

canon 修改后必须全文检索该名词（含别名与拼写漂移，见 0.2 GLOSSARY）所有出现处并同步；改完跑 `tools/lint-book.sh --write`。
反面教材（这条规则的由来）：若改 Riverbend 地理时不查水系 canon，极易把"南支自 Orun 北壁下行、两支汇于盆地中部"（0.2 / 2.1）写成"河从北向南流进沙漠"。读后才答、改后必扫，就是防这个。

## 五、过审门与账本

- 动 canon：过 **6.12 canon-change-gate**；扫**第 5 章三本账**（连续性 / 意象链 / 伏笔）并修账。
- 写 prose：过 **6.11 prose-gate**；自检中"发现并修复"的失败，前后两版 + 一行原因进**第 9 章配对样本**。
- 派生物方向永远是 **大文件 → 派生物**（0.4 外部引用清单由脚本生成；第 0 章索引与正文冲突时以正文为准、并修索引）。

## 六、AI 如何知道自己不知道（认知协议，自旧 README 迁入）

**公理：印象不是出处。** 检索式 AI 对本世界的默认状态是"不知道"——预训练里没有 Archeon，聊天记忆会过期，90-archive 的标本会污染印象。任何 canon 断言的唯一合法来源是大文件原文。

**断言三态制**：每一个世界事实断言必须归入三态——
1. **可溯源**：能当场给出大文件章节定位 → 直接使用；动 canon 的 PR 须附出处。
2. **待查**：说不出位置，但索引/GLOSSARY 提示存在 → 先查后写（grep 大文件 / 派 agent 读）；查到升为可溯源，查无降为真空白。
3. **真空白**：大文件与索引均无 → 见下。禁止脑补填平。

判定就一句话："我能给出章节定位吗？"给不出 = 没读过 = 待查。没有第四态。

**写前自检**：
- 本段每个专名（人/地/物/事件）是否逐一对过 0.2 GLOSSARY 拼写与正文定义？
- 角色的年龄、性别、家庭关系、生死状态，是否有 1.7 Characters 的明文出处？**不知道就去查，查不到就问，不许编，可拒答**（作者法条）。
- 我引用的是大文件原文，还是对历史对话的印象？后者必须回卷验证——聊天记忆可能停在被推翻的旧版。
- 读过 90-archive 标本的，警惕标本污染（实证：旧稿 Nakamura 为女性，canon 为男性老园丁；旧稿 Axiom 在轨五百年，canon 为 ~2506 坠毁）。
- 用到的条款是 canon 还是 proposed？proposed 一律注明"待裁决"，不得当事实铺陈（如 Ring，见 6.7）。

**真空白的处理**（空白即资产，任何一条都不允许"顺手编一个填进正文"）：
- 局部小空白：正文绕开，或以 proposed 显式提案标【※】，进收割清单等裁决。
- 结构性空白：挂入 7.2 待决问题清单追加编号，正文暂以不依赖该事实的方式写。
- 作者意图级空白：直接问作者，等一句话裁决。宁可产出减半，不可埋一颗矛盾。

**禁令清单**：
- 禁止用预训练知识补本世界设定（地球通识可用于物理/气象等现实机制，Archeon 专有事实必须有卷内出处）。
- 禁止从聊天记忆直接引 canon——记忆只用于"知道该去读哪一章"。
- 禁止把"合理推断"写成事实；推断必须显式标注为推断或【※】。
- 禁止在不确定时选择"更流畅的那个版本"——流畅是编造的伪装色。不确定的正确形态是难看的：停下来，查，或问。
- 禁止默写专名拼写；一律对 0.2 GLOSSARY。

**session 末的无知申报**：收割清单（6.13）末尾追加"本次发现的不知道"：列出本 session 的待查项与真空白及处置（已查清/已挂账/已问作者）。空白被登记，下一个 session 才不会在同一个坑里重新猜。

> 知道自己不知道，是检索式接班人唯一可靠的美德。
